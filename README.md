# POP-SPOT

서울에서 매주 새로 열리는 팝업스토어를 **한 화면에 모아주는 서비스**.

**[popspot.co.kr](https://popspot.co.kr/)** 에서 직접 써볼 수 있어요.

</div>

---

## 어떤 서비스인가요?

성수동·한남·압구정 같은 서울의 팝업스토어는 매주 새로 열리고 또 사라져요. 인스타그램 수십 개 계정을 팔로우해야 겨우 따라잡을 수 있죠. POP-SPOT 은 그 정보를 **한 곳에 모아서**, 지도와 캘린더로 한눈에 보여줍니다.

이런 사람들에게 좋아요:

- 팝업 투어를 자주 다니는데 매번 인스타에서 찾는 게 지치는 분
- 오늘 데이트 코스를 짜야 하는데 어디 갈지 모르겠는 분
- 친구랑 같이 팝업 다니기로 했는데 동선 짜기가 귀찮은 분
- 다녀온 팝업을 기록으로 남기고 싶은 분

## 어떻게 시작해요?

**가입 안 하고 둘러보기** — 회원가입이 부담스러우면 7일 동안 무료로 둘러볼 수 있어요. 로그인 화면의 「게스트로 7일 둘러보기」 버튼을 누르세요.

- 7일 동안: 지도 / 캘린더 / 마이페이지 / 도장 등 대부분 기능 사용 가능
- AI 코스 추천 / 노래로 추천 / 친구 동선 기능은 정식 회원만 사용 가능
- 7일이 지나면 자동으로 회원가입 안내가 떠요

**바로 회원가입** — 카카오 / 네이버 / 구글 계정으로 한 번에. 이메일 가입도 가능.

## 팝업 정보는 어떻게 모으나요?

매일 새벽 4시에 컴퓨터가 자동으로 일을 해요:

1. **검색** — 네이버와 카카오에 "성수 팝업", "한남 팝업" 같은 키워드 60개로 검색.
2. **정리** — 검색 결과에서 AI 가 가게 이름·위치·기간을 뽑아냅니다.
3. **확인** — AI 가 "이건 진짜 팝업이다" 라고 80% 이상 확신하면 자동으로 사이트에 올리고, 아니면 사람이 한 번 더 검토할 수 있게 대기열로.
4. **만료 처리** — 매일 새벽 5시에 끝난 팝업을 자동으로 "종료" 로 표시.

> 인터넷에서 글 본문을 통째로 긁어오지 않고, 검색 결과가 보여주는 제목·짧은 설명·링크만 사용해요. 저작권 위반이 안 되게 설계.

이 과정에서 지키는 7가지 약속:

- 공식 검색 API 만 사용 (스크래핑 X)
- 사람 이름·휴대폰·이메일 같은 개인정보는 AI 한테 안 보냄
- 잘못된 정보 신고는 24시간 안에 처리
- "AI 가 정리한 정보" 라는 안내를 항상 같이 표시
- 끝난 팝업은 매일 자동으로 정리
- 약관과 정책을 사이트 안에서 누구나 볼 수 있게
- 가입 시 약관 동의 체크

## 노래로 팝업 추천받기

특이한 기능 하나 — 좋아하는 노래를 클릭하면 그 곡에 어울리는 팝업 5개를 골라줘요.

```
좋아하는 노래 선택  →  AI 가 노래의 분위기 분석  →  분위기와 어울리는 팝업 추천
```

예를 들어 잔잔한 발라드를 고르면 카페 / 전시 위주의 팝업을, 신나는 K-POP 을 고르면 패션 / 푸드 팝업을 추천하는 식이에요. 노래는 유튜브에서 바로 재생해 들으면서 추천 팝업을 둘러볼 수 있어요.

## 개발 환경 설정

### 필요한 것

| 도구 | 버전 |
|---|---|
| JDK | 21 (Temurin 권장) |
| Node.js | 20 LTS |
| PostgreSQL | 14 이상 |
| Redis | 6 이상 |

### 백엔드 (서버) 실행

```bash
git clone https://github.com/hanshhx/pop-spot.git
cd pop-spot/popspot-backend

# 1. 환경 변수 채우기
cp .env.example .env.local
# .env.local 을 열어서 본인 API 키·비밀번호 입력

# 2. PostgreSQL / Redis 가 실행 중인지 확인
#    macOS:  brew services start postgresql@14 redis
#    Ubuntu: sudo systemctl start postgresql redis-server

# 3. 서버 실행
./run-local.ps1                                     # Windows
set -a && source .env.local && set +a && ./gradlew bootRun  # macOS / Linux
```

서버는 `localhost:8080` 에서 뜸. 비밀번호 같은 필수 환경 변수가 비어 있으면 의도적으로 안 켜져요.

### 프론트엔드 (웹 화면) 실행

```bash
cd pop-spot/popspot-frontend
cp .env.example .env.local
npm install
npm run dev   # http://localhost:3000
```

## 환경 변수 (백엔드)

값을 절대 깃에 커밋하지 마세요. 운영 서버에서는 시스템 환경 변수로, Vercel 에서는 프로젝트 설정에서 주입.

| 키 | 무엇에 쓰여요 |
|---|---|
| `DB_URL` · `DB_USERNAME` · `DB_PASSWORD` | PostgreSQL 데이터베이스 접속 정보 |
| `JWT_SECRET` | 로그인 토큰 서명용 비밀 키 (32 글자 이상 필수) |
| `JWT_EXPIRATION` | 로그인 유지 시간 (밀리초, 기본 1시간) |
| `MAIL_USERNAME` · `MAIL_PASSWORD` | 이메일 인증 발송용 Gmail 앱 비밀번호 |
| `GROQ_API_KEY` · `GROQ_MODEL_NAME` · `GROQ_BASE_URL` | AI 호출용 키 / 모델 / 주소 |
| `NAVER_CLIENT_ID` · `NAVER_CLIENT_SECRET` | 네이버 검색 API + 네이버 로그인 키 |
| `KAKAO_REST_API_KEY` · `KAKAO_LOCAL_API_KEY` | 카카오 검색 / 주소→좌표 변환 키 |
| `SPOTIFY_CLIENT_ID` · `SPOTIFY_CLIENT_SECRET` | 스포티파이 노래 검색 키 |
| `YOUTUBE_API_KEY` | 유튜브 영상 검색 키 |
| `SENTRY_DSN` | 에러 자동 보고용 (운영에서만) |
| `APP_ALLOWED_ORIGINS` | 어떤 도메인이 백엔드를 호출할 수 있는지 화이트리스트 |
| `OAUTH_*_CLIENT_ID/SECRET` | 구글 / 카카오 / 네이버 소셜 로그인 키 |

## 환경 변수 (프론트엔드)

## 배포 환경

| 부분 | 어디에 |
|---|---|
| 웹 화면 | Vercel — `main` 에 머지하면 자동 배포 · 도메인 `popspot.co.kr` |
| 서버 | 친구네 Synology NAS 위 가상머신 (Ubuntu 22.04) |
| 안전한 통로 | Tailscale Funnel — 인증서 자동 갱신, 별도 nginx 없음 |
| 데이터베이스 | 동일 가상머신 안에 PostgreSQL + Redis |
| 자동 작업 | 새벽 4시 팝업 자동수집, 5시 만료 처리 |
| DB 변경 관리 | Flyway — 새 컬럼 추가 등 변경을 코드로 관리 |

자세한 운영 명령은 [`SYNOLOGY_MIGRATION_GUIDE.md`](./SYNOLOGY_MIGRATION_GUIDE.md), 전체 변경 이력은 [`PROJECT_CHANGELOG.md`](./PROJECT_CHANGELOG.md) 참고.

## 자주 막혔던 문제들

| # | 어떤 문제 | 어떻게 해결 |
|:--:|---|---|
| 1 | 채팅 메시지 저장이 실패 | 데이터베이스 종류를 Oracle → PostgreSQL 로 바꿔서 ID 자동 생성으로 해결 |
| 2 | "아이유 좋은날" 을 스포티파이가 못 찾음 | AI 로 "IU good day" 영문 변환 후 재검색 |
| 3 | API 키가 깃에 노출돼서 자동 차단됨 | 시크릿 검사 자동 스크립트 추가 |
| 4 | 무료 회원이 코스를 무제한 저장하던 버그 | 비동기 콜백 안의 return 이 외부 함수를 종료 안 시키는 함정. await + return 으로 수정 |
| 5 | 빌드 자동 검사가 한국어 주석을 못 맞춰 실패 | 주석을 짧은 문장으로 다시 작성 + 라인 길이 100 칸 안에 맞춤 |
